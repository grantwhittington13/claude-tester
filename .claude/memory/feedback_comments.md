---
name: Use comments sparingly
description: Only add code comments when logic is genuinely complex or non-obvious
type: feedback
---

Only comment complex code. Do not add comments to self-evident logic, obvious variable names, or standard patterns.

**Why:** User preference — comments should add real value, not narrate what the code already says clearly.

**How to apply:** Before adding any comment, ask whether the code is unclear without it. Skip comments on straightforward assignments, simple conditionals, and standard library usage.
